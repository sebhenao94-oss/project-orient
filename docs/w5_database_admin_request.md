# W5 database administrator request

Project ORIENT's `orient_team_4` role can already `SELECT`, `INSERT`, and
`UPDATE` `public.equipment_details`. We verified read-only that
`equipment_details.equipment_id` is `GENERATED BY DEFAULT AS IDENTITY`, so the
identity column allocates IDs on `INSERT` with **no sequence grant required** —
the earlier `GRANT ... ON SEQUENCE …equipment_id_seq` ask is not needed.

The only thing the team cannot do is create the W5 review tables (it has no
`CREATE` on the `public` schema). Please perform the following as a database
owner/administrator:

1. Review and apply `pipeline/review_schema.sql` to create:
   - `public.review_session`
   - `public.review_action`
   - `public.correction_log`
2. Grant the runtime DML privileges below on those three tables.

```sql
BEGIN;

GRANT SELECT, INSERT, UPDATE
ON TABLE public.review_session,
         public.review_action,
         public.correction_log
TO orient_team_4;

COMMIT;
```

No `CREATE` on the `public` schema and no sequence grant are requested. The
application must not hand-allocate `equipment_id` values — the identity column
assigns them.

## Verification queries

Run as `orient_team_4` after the change; all results should be `true`:

```sql
SELECT
    to_regclass('public.review_session') IS NOT NULL AS review_session_exists,
    to_regclass('public.review_action')  IS NOT NULL AS review_action_exists,
    to_regclass('public.correction_log') IS NOT NULL AS correction_log_exists,
    has_table_privilege(current_user, 'public.review_session', 'INSERT')
        AS can_insert_sessions;
```
